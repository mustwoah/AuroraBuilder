## ğŸš€ How It Works

Aurora Builder leverages **GitHub Actions** as a cloud build server. Here is the magic happening behind the scenes:

1.  **Configuration**: You design your app in the Aurora web interface.
2.  **Generation**: The browser generates a complete **Flutter** project (`main.dart`, `pubspec.yaml`, etc.) in memory.
3.  **Push**: Using your GitHub Token, the app creates a temporary repository on your account and pushes the code.
4.  **Build**: A GitHub Action workflow is triggered automatically to compile the APK.
5.  **Deliver**: The web interface polls the API and lets you download the artifact directly.
